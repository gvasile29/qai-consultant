# Stack Overflow 2025 Developer Survey: AI Section

## Section 3: AI

### 3.1 Sentiment and Usage

#### AI Tools in Development Process

Adoption of AI tools continues to rise, with **84% of respondents using or planning to use AI in development**, up from 76% the previous year. Among professional developers, **51% use AI tools daily**.

**Usage breakdown (All Respondents):**
- Daily use: 47.1%
- Weekly use: 17.7%
- Monthly/infrequent use: 13.7%
- Plan to use soon: 5.3%
- No plans to use: 16.2%

**Professional developers show highest daily adoption at 50.6%**, while those learning to code lag at 39.5%. Early-career developers (1-5 years experience) lead with **55.5% daily usage**.

#### AI Tool Sentiment Decline

Positive sentiment has declined significantly. "70%+ favorable in 2023-2024 dropped to 60% in 2025." Professional developers maintain higher favorable sentiment (61%) compared to learners (53%).

**Sentiment breakdown (All Respondents):**
- Very favorable: 22.9%
- Favorable: 36.8%
- Indifferent: 17.6%
- Unsure: 2.3%
- Unfavorable: 10.8%
- Very unfavorable: 9.6%

### 3.2 Developer Tools

#### Accuracy Concerns

Developer trust in AI accuracy remains low. **46% actively distrust accuracy while 33% trust it**, with only 3% reporting high trust levels. Experienced developers (10+ years) show the most caution, with the lowest "highly trust" rate at 2.6%.

**Trust distribution (All Respondents):**
- Highly trust: 3.1%
- Somewhat trust: 29.6%
- Somewhat distrust: 26.1%
- Highly distrust: 19.6%

#### Handling Complex Tasks

Improvements noted: **29% of professional developers believe AI struggles with complex tasks**, down from 35% in 2024. Overall, only 4.4% report AI handling complex tasks very well.

**Competency assessment (All Respondents):**
- Very well: 4.4%
- Good but not great: 25.2%
- Neither good nor bad: 14.1%
- Bad: 22%
- Very poor: 17.6%
- Don't use for complex tasks: 16.8%

#### Development Workflow Integration

Developers show strongest acceptance for information retrieval tasks and greatest resistance to high-stakes operations:

**Currently using mostly AI:**
- Search for answers: 54.1%
- Generating synthetic data: 35.8%
- Learning concepts: 33.1%
- Code documentation: 30.8%

**Highest rejection (Don't plan to use):**
- Deployment/monitoring: 75.8%
- Project planning: 69.2%
- Code review: 58.7%

#### Key Frustrations

The biggest challenge developers face is **"AI solutions that are almost right, but not quite" (66%)**. This leads to the second-largest frustration: **"Debugging AI-generated code is more time-consuming" (45%)**.

Other frustrations include:
- Reduced personal confidence in problem-solving: 20%
- Difficulty understanding code logic: 16.3%
- Other issues: 11.6%
- No problems encountered: 4%

#### Future Reliance on Human Developers

Despite AI advancement, **75% would still seek help when they "don't trust AI's answers."** Other reasons developers want human assistance:
- Ethical/security concerns: 61.7%
- Full understanding desired: 61.3%
- Learning best practices: 58.1%
- Complex problem-solving: 49.8%

#### Vibe Coding Adoption

"Vibe coding" (generating software from LLM prompts) remains niche. **72% report it's not part of their workflow**, with 5.3% emphatic about rejecting it. Only 11.9% actively practice it.

### 3.3 AI Agents

#### Current Agent Usage

**AI agents remain mainstream adjacent.** 52% either don't use agents or stick to simpler AI tools, while **38% have no plans to adopt them**.

**Usage patterns (All Respondents):**
- Daily use: 14.1%
- Weekly use: 9%
- Monthly/infrequent use: 7.8%
- Plan to use: 17.4%
- Using copilot/autocomplete only: 13.8%
- No plans: 37.9%

Professional developers show slightly higher adoption (**14.9% daily**), while those learning report lower adoption (13.2% daily).

#### Productivity Impact

**51.6% agree AI tools/agents positively impact productivity** (combining "great extent" and "somewhat"):
- Great extent: 16.3%
- Somewhat: 35.3%
- Not at all/minimally: 41.4%

#### Industry Applications

Among developers using agents:
- **Software engineering: 83.5%** (dominant use case)
- Data/analytics: 24.9%
- IT operations: 18%
- Business automation: 17.6%

**General purpose tasks** (agents used outside development):
- Language processing: 49%
- External API integration: 38.3%
- MCP servers: 34.4%
- Multi-agent orchestration: 28.1%

#### Agent Impact Assessment

Developers report personal efficiency gains but limited team collaboration improvement:
- Reduced task time: 68.7% agreement
- Increased productivity: 68.8% agreement
- Quality improvement: 37.5% agreement
- **Team collaboration improvement: Only 17.3%** (significant gap)

#### Agent Challenges

Major concerns center on reliability and security:
- **Accuracy concerns: 86.9% agree** (combining "strongly" and "somewhat")
- **Security/privacy concerns: 81.4% agree**
- Integration difficulty: 46.2% agree
- Significant learning curve: 43.4% agree
- Restrictive IT policies: 28.4% agree
- Cost barriers: 25.3% agree

#### Data Management Tools

Developers repurpose traditional tools for AI agent memory:
- Redis: 42.9%
- GitHub MCP Server: 42.8%
- Supabase: 20.9%
- ChromaDB: 19.7%
- pgvector: 17.9%

#### Agent Orchestration Frameworks

Open-source tools dominate:
- Ollama: 51.1%
- LangChain: 32.9%
- LangGraph: 16.2%
- Vertex AI: 15.1%
- Amazon Bedrock: 14.5%

#### Observability and Security Tools

Developers adopt existing DevOps tools rather than AI-native solutions:
- Grafana + Prometheus: 43%
- Sentry: 31.8%
- Snyk: 18.2%
- New Relic: 13%
- LangSmith: 12.5%

#### Popular Out-of-Box Solutions

Market leaders for agent assistance:
- ChatGPT: 81.7%
- GitHub Copilot: 67.9%
- Google Gemini: 47.4%
- Claude Code: 40.8%
- Microsoft Copilot: 31.3%

---

## Key Takeaways

1. **Adoption continues despite sentiment decline** — Usage grows while satisfaction drops, suggesting people use AI tools despite reservations

2. **Trust remains the critical gap** — Nearly half of developers distrust AI accuracy, driving continued verification requirements

3. **AI agents remain specialized** — Not mainstream; most developers stick to copilot/autocomplete modes rather than autonomous agents

4. **Team collaboration lags** — Despite individual productivity claims, agents show minimal impact on team-level collaboration

5. **High-stakes tasks remain human-dependent** — Deployment, planning, and review tasks show strong resistance to AI automation
